# Writing Plans: PM Action Planning Guide

Transform approved designs into a series of verifiable, traceable tasks.

## Core Mandates

- **Don't Skip Verification**: Any plan that includes "implementation" must first include "verification."
- **Atomic Tasks**: Each task should have a single output (e.g., an interview outline, a metric definition).
- **User Review**: You must have the user confirm the plan before starting task execution.

## Checklist

1. **Analyze Spec**: Ensure full understanding of the logic in `docs/pmpowers/specs/`.
2. **Develop Phased Tasks**:
   - **Phase 1: Discovery/Validation**: De-risking.
   - **Phase 2: Definition**: Detailed logic.
   - **Phase 3: Final Guard**: End-to-end verification.
3. **Set Checkpoints**: Pause at critical decision points to get user feedback.
4. **Save Plan**: Write to the temporary plan directory and execute after approval.

## Verification & Traceability

Every task in the plan must be linked to the North Star Metric. If a task cannot explain how it improves the core metric, it is waste.
