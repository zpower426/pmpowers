# Spec Document Reviewer Prompt Template

使用此模板派发产品规格说明书（Spec）评审子代理。

**目的:** 验证 Spec 是否完整、一致，并准备好进入实施计划阶段。

**派发时机:** Spec 文档已写入 `docs/pmpowers/specs/` 之后。

```
Task tool (general-purpose):
  description: "Review spec document"
  prompt: |
    你是一名产品规格说明书评审专家。请验证此 Spec 是否完整并准备好进行计划。

    **待评审的 Spec:** [SPEC_FILE_PATH]

    ## 检查要点

    | 类别 | 检查内容 |
    |----------|------------------|
    | 完整性 | TODO、占位符、"TBD"、不完整的章节 |
    | 一致性 | 内部逻辑矛盾、相互冲突的需求 |
    | 清晰度 | 需求是否模糊到足以导致执行偏差 |
    | 范围 | 是否足够聚焦于单一计划 — 未涵盖多个独立的子系统 |
    | 精益 (YAGNI) | 未经请求的功能、过度设计 |

    ## 评审准则

    **仅指出会导致实施计划出现真实问题的缺陷。**
    缺失章节、逻辑矛盾、或者足以产生两种歧义的需求 —— 这些是问题。
    微小的措辞改进、风格偏好、或者“某些部分不够详细”不属于阻碍性问题。

    除非存在会导致计划失败的严重差距，否则请予以批准。

    ## 输出格式

    ## Spec 评审报告

    **状态:** 已批准 | 发现问题

    **发现的问题 (如果有):**
    - [章节 X]: [具体问题] - [为什么它会影响计划执行]

    **改进建议 (非阻碍性):**
    - [改进建议]
```

**评审返回:** 状态、问题清单 (如果有)、改进建议
