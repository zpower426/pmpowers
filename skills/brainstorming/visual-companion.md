# Visual Companion Guide (Pmpowers Edition)

基于浏览器的视觉脑暴伴侣，用于展示原型、图表和产品方案。

## 何时使用

针对每个问题独立决定，而非整个会话。测试标准：**看到它是否比阅读它更能让用户理解？**

**使用浏览器**：当内容本质上是视觉的时：
- **产品原型 (Mockups)** — 页面布局、导航结构、交互逻辑。
- **机会地图与指标树** — OST、北极星指标层级图。
- **方案对比** — 侧重于视觉布局或流程流转的 A/B 对比。
- **状态机与流程图** — 复杂的业务逻辑可视化。

**使用终端**：当内容是文本或表格时：
- **需求与范围问题** — “这个功能包含什么？”。
- **概念性选择** — 纯逻辑上的方案挑选。
- **权衡列表** — 优缺点对比。

## 启动会话

```bash
# 启动服务器并持久化 mockups 到项目目录
scripts/start-server.sh --project-dir /path/to/project
```

## 设计准则

- **保真度对齐** — 脑暴阶段使用低保真原型，定义阶段使用高保真逻辑图。
- **每次都解释问题** — 明确当前视觉呈现的目的是什么。
- **2-4 个选项** — 避免认知负荷。

## 文件命名规范
- 使用语义化名称：`opportunity-map.html`, `user-flow.html`, `spec-logic.html`。
- **永远不要重用文件名** — 每个新屏幕都必须是一个新文件。
- 服务器会自动通过修改时间提供最新的文件。
