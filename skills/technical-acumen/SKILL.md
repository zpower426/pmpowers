# Technical Acumen: Technical Sensitivity and Architectural Understanding

Understanding technology is for better business trade-offs.

## Checklist

1. **Identify Technical Constraints** — Does the existing architecture support this feature? Is refactoring required?
2. **Analyze Interface Contracts (API Design)** — Do input parameters, output parameters, and error codes align with business logic?
3. **Assess Implementation Cost (Estimation)** — Is this a "one-day" task or a "one-month" engineering effort?
4. **Identify Technical Debt Risks** — What long-term scalability have we sacrificed to meet a deadline?

## Core Framework: Technical Assessment Model

- **Scalability**: Will the system crash if the user base grows 10x?
- **Security**: How is user privacy data encrypted? Are there injection risks?
- **Consistency**: In case of network jitter, will data become corrupted or inconsistent?

## Expert Principles

Before proposing a requirement to R&D, ask yourself: "If this were my own money, how much would I be willing to spend to implement this feature?"
